            <h2 id="install" tabindex="-1">Install <a href="#install" aria-hidden="true" data-is-anchor="true">#</a></h2>

          
      
          
            
            
            
            
            
              
              
            
            <details><summary role="button" class="outline"><b>Install HTML blades</b> via npm</summary>
<pre><code class="language-sh">npm install @anydigital/blades
cd ./_includes  # your includes dir
ln -s ../node_modules/@anydigital/blades/_includes/blades
</code></pre>
<p>That's it! Now you can use HTML blades in your templates like this:</p>
<p><mark>In Nunjucks:</mark></p>
<pre><code class="language-jinja2">{% extends 'blades/html.njk' %}
{% include 'blades/gtm.njk' %}
</code></pre>
<p><mark>In Liquid:</mark></p>
<pre><code class="language-liquid">{% include blades/html.liquid %}
{% include blades/gtm.liquid for_body=true %}
{% render blades/links, links: site.links, current_url: page.url %}
</code></pre>
</details>
<hr>
<p>All CSS and HTML blades above are available as a Jekyll theme:</p>
<details><summary role="button" class="outline"><b>Install as Jekyll theme</b></summary>
<p>In you <code>_config.yml</code>:</p>
<pre><code class="language-yaml">remote_theme: anydigital/blades@v0.27.0-beta # or latest
plugins:
  - jekyll-remote-theme
</code></pre>
<p>Living example: <a href="https://github.com/anydigital/bladeswitch/blob/main/_config.yml"><i><img src="https://www.google.com/s2/favicons?domain=github.com&sz=64"></i> /anydigital/bladeswitch/blob/main/_config.yml</a></p>
</details>
<p>Or use a preconfigured template:</p>
<p><a href="https://github.com/anydigital/bladeswitch" role="button" class="outline">🥷 Bladeswitch Starter ↗  <small style="white-space: nowrap">Jekyll ⁺ Pico ⁺ Blades</small></a></p>

<h2 id="base" tabindex="-1">Base HTML <a href="#base" aria-hidden="true" data-is-anchor="true">#</a></h2>

          
      
          
            
            
            
            
            
              
              
            
            <p>Usage in Nunjucks layout:</p>
<pre><code class="language-jinja2">{% extends 'blades/html.njk' %}

{% block body %}...{% endblock %}
</code></pre>
<p>Living example: <a href="https://github.com/anydigital/build-awesome-starter/blob/main/_includes/default.njk"><i><img src="https://www.google.com/s2/favicons?domain=github.com&sz=64"></i> /anydigital/build-awesome-starter/blob/main/_includes/default.njk</a></p>
<br>
<details class="breakout-item"><summary role="button" class="outline">How it works</summary>
<pre><code class="language-jinja2">&lt;!doctype html&gt;
&lt;html lang=&quot;{{ site.lang | d('en') }}&quot;&gt;
  &lt;head&gt;
    &lt;meta charset=&quot;utf-8&quot; /&gt;
    &lt;meta name=&quot;viewport&quot; content=&quot;width=device-width, initial-scale=1, shrink-to-fit=no, viewport-fit=cover&quot; /&gt;
    &lt;link rel=&quot;icon&quot; href=&quot;{{ site.favicon | d('/favicon.ico') }}&quot; /&gt;
    &lt;title&gt;
      {{- title | string | striptags ~ ' | ' if title -}}
      {{- site.title -}}
    &lt;/title&gt;
    &lt;meta name=&quot;description&quot; content=&quot;{{ summary }}&quot; /&gt;

    {%- for href in site.styles %}
      &lt;link rel=&quot;stylesheet&quot; href=&quot;{{ href }}&quot; /&gt;
    {%- endfor %}
    &lt;style&gt;
      {{ site.inline_styles | d([]) | join('\n') }}
    &lt;/style&gt;

    {%- for src in site.scripts %}
      &lt;script src=&quot;{{ src }}&quot; defer&gt;&lt;/script&gt;
    {%- endfor %}
    &lt;script type=&quot;module&quot;&gt;
      {{ site.inline_scripts | d([]) | join('\n') }}
    &lt;/script&gt;

    {{ content_for_header }}
    {% include 'blades/gtm.njk' %}
  &lt;/head&gt;

  &lt;body&gt;
    {% set for_body = true %}{% include 'blades/gtm.njk' %}
    {% block body %}
    {% endblock %}
  &lt;/body&gt;
&lt;/html&gt;
</code></pre>
<ul>
<li><code>title | string</code> avoids error with <code>| striptags</code> when you pass a pure number.</li>
</ul>
</details>

          
      
          
            
            
            
            
            
              
              
            
            <p>Usage in Liquid layout:</p>
<pre><code class="language-liquid">{% capture body %}...{% endcapture %}

{% include blades/html.liquid %}
</code></pre>
<p>Living example: <a href="https://github.com/anydigital/bladeswitch/blob/main/_includes/default.liquid"><i><img src="https://www.google.com/s2/favicons?domain=github.com&sz=64"></i> /anydigital/bladeswitch/blob/main/_includes/default.liquid</a></p>
<br>
<details class="breakout-item"><summary role="button" class="outline">How it works</summary>
<pre><code %="" endcomment="" %="" class="language-liquid">&lt;!doctype html&gt;
&lt;html lang=&quot;{{ site.lang | default: 'en' }}&quot;&gt;
  &lt;head&gt;
    &lt;meta charset=&quot;utf-8&quot;&gt;
    &lt;meta name=&quot;viewport&quot; content=&quot;width=device-width, initial-scale=1, shrink-to-fit=no, viewport-fit=cover&quot;&gt;
    &lt;link rel=&quot;icon&quot; href=&quot;{{ site.favicon | default: '/favicon.ico' }}&quot;&gt;
    &lt;title&gt;
      {%- if title %}{{ title | strip_html | append: ' | ' }}{% endif -%}
      {{- site.title -}}
    &lt;/title&gt;
    &lt;meta name=&quot;description&quot; content=&quot;{{ summary }}&quot;&gt;

    {%- for href in site.styles %}
      &lt;link rel=&quot;stylesheet&quot; href=&quot;{{ href | relative_url }}&quot;&gt;
    {%- endfor %}
    &lt;style&gt;
      {{ site.inline_styles | join: '\n' }}
    &lt;/style&gt;

    {%- for src in site.scripts %}
      &lt;script src=&quot;{{ src }}&quot; defer&gt;&lt;/script&gt;
    {%- endfor %}
    &lt;script type=&quot;module&quot;&gt;
      {{ site.inline_scripts | join: '\n' }}
    &lt;/script&gt;

    {{ content_for_header }}
    {% include blades/gtm.liquid %}
  &lt;/head&gt;

  &lt;body&gt;
    {% include blades/gtm.liquid for_body=true %}

    {{ body }}
  &lt;/body&gt;
&lt;/html&gt;
</code></pre>
</details>

<h2 id="gtm" tabindex="-1">Google Tag Manager (GTM) <a href="#gtm" aria-hidden="true" data-is-anchor="true">#</a></h2>

          
      
          
            
            
            
            
            
              
              
            
            <p>Usage in Nunjucks:</p>
<pre><code class="language-jinja2">    ...
    {% include 'blades/gtm.njk' %}
  &lt;/head&gt;
  &lt;body&gt;
    {% set for_body = true %}{% include 'blades/gtm.njk' %}
    ...
</code></pre>
<p>Living example: <a href="https://github.com/anydigital/blades/blob/main/_includes/blades/html.njk"><i><img src="https://www.google.com/s2/favicons?domain=github.com&sz=64"></i> /anydigital/blades/blob/main/_includes/blades/html.njk</a></p>
<p>Parameters:</p>
<ul>
<li><code>site.gtm_id</code> - Your Google Tag Manager container ID (e.g., <code>GTM-XXXXXXX</code>)</li>
<li><code>site.prod</code> - Boolean flag to enable GTM only in production</li>
<li><code>for_body</code> - Boolean flag (default: <code>false</code>). When <code>false</code>, renders the script tag for the <code>&lt;head&gt;</code>. When <code>true</code>, renders the noscript fallback for the <code>&lt;body&gt;</code>.</li>
</ul>
<br>

<h5 id="njk-vscode" tabindex="-1">Syntax highlighting <sub>in <a href="/tricks/antigravity/">VS Code~editors</a></sub> <a href="#njk-vscode" aria-hidden="true" data-is-anchor="true">#</a></h5>
<ul>
<li>🧩 Install <a href="https://github.com/edheltzel/better-nunjucks-for-visual-studio-code"><i><img src="https://www.google.com/s2/favicons?domain=github.com&sz=64"></i> /edheltzel/better-nunjucks-for-visual-studio-code</a></li>
</ul>
<p>This is a modern fork of the original extension. It is specifically designed to solve the "11ty problem" where you mix Nunjucks and HTML.</p>
<p><strong>Why it's better:</strong> It injects Nunjucks grammar directly into the standard HTML grammar. This means you get full Nunjucks highlighting and the editor still knows it's an HTML file, giving you better Emmet and CSS autocompletion.</p>
<p><strong>No Config:</strong> It works out of the box without needing to manually map file associations in your settings.</p>
<h5 id="auto-formatting-in-vs-code-editors" tabindex="-1">Auto-formatting <sub>in <a href="/tricks/antigravity/">VS Code~editors</a></sub> <a href="#auto-formatting-in-vs-code-editors" aria-hidden="true" data-is-anchor="true">#</a></h5>
<ol>
<li>
<p>🧩 Install <a href="https://github.com/prettier/prettier-vscode"><i><img src="https://www.google.com/s2/favicons?domain=github.com&sz=64"></i> /prettier/prettier-vscode</a> (if not yet)</p>
</li>
<li>
<p>Install a compatible Prettier plugin for your project, for example:</p>
</li>
</ol>
<pre><code class="language-sh">npm i -D prettier-plugin-jinja-template
</code></pre>
<ol start="3">
<li>It might be tricky to find a well-maintained Nunjucks plugin, but <code>jinja-template</code> works just fine with <code>.njk</code> via <code>.html</code> override:</li>
</ol>
<pre><code data-caption=".prettierrc.json" class="language-json">{
  &quot;plugins&quot;: [&quot;prettier-plugin-jinja-template&quot;],
  &quot;overrides&quot;: [
    {
      &quot;files&quot;: [&quot;*.njk&quot;, &quot;*.html&quot;],
      &quot;options&quot;: {
        &quot;parser&quot;: &quot;jinja-template&quot;
      }
    }
  ]
}
</code></pre>
<p><code>PRO TIP:</code> If you already use <a href="https://github.com/anydigital/blades"><i><img src="https://www.google.com/s2/favicons?domain=github.com&sz=64"></i> /anydigital/blades</a>, it’s even easier. You can simply:</p>
<pre><code class="language-sh">ln -s ./node_modules/@anydigital/blades/.prettierrc.json
</code></pre>
<h5 id="sort-array-by-attribute" tabindex="-1">Sort array by attribute <a href="#sort-array-by-attribute" aria-hidden="true" data-is-anchor="true">#</a></h5>
<p>Per official <code>.njk</code> documentation:</p>
<blockquote>
<p><strong>sort(arr, reverse, caseSens, attr)</strong><br>
Sort arr with JavaScript's arr.sort function. If reverse is true, result will be reversed. Sort is case-insensitive by default, but setting caseSens to true makes it case-sensitive. If attr is passed, will compare attr from each item.</p>
</blockquote>
<p>But you can actually do this trick:</p>
<pre><code data-caption=".njk" class="language-jinja2">{% for item in array | sort(attribute='weight') %}
  ...
{% endfor %}
</code></pre>
<h5 id="include-and-render-md-file-w-o-its-front-matter-in-11ty" tabindex="-1">Include and render <code>.md</code> file w/o its Front Matter <sub>in <code>11ty</code></sub> <a href="#include-and-render-md-file-w-o-its-front-matter-in-11ty" aria-hidden="true" data-is-anchor="true">#</a></h5>
<pre><code data-caption=".njk" class="language-jinja2">{# first, get the raw content using `html` as plain-text engine #}
{% set _eval = &quot;{% renderFile './YOUR_FILE.md', {}, 'html' %}&quot; %}
{% set _raw_md = _eval | renderContent('njk') %}

{# then, remove the front matter using regex, and render using `md` #}
{{ _raw_md | replace(r/^---[\s\S]*?---/, '') | renderContent('md') | safe }}
</code></pre>

<h5 id="beware-false-positives-in-liquid" tabindex="-1">Beware: False Positives <small>in <code>.liquid</code></small> <a href="#beware-false-positives-in-liquid" aria-hidden="true" data-is-anchor="true">#</a></h5>
<ul>
<li><a href="https://liquidjs.com/tutorials/truthy-and-falsy.html"><i><img src="https://www.google.com/s2/favicons?domain=liquidjs.com&sz=64"></i> /tutorials/truthy-and-falsy.html</a></li>
<li><a href="https://www.11ty.dev/docs/languages/liquid/#java-script-truthiness-in-liquid"><i><img src="https://www.google.com/s2/favicons?domain=www.11ty.dev&sz=64"></i> /docs/languages/liquid/#java-script-truthiness-in-liquid</a></li>
</ul>
<p>❌ Wrong:</p>
<pre><code class="language-liquid">{% if my_string %} This will always show if the string exists, even if empty. {% endif %}
</code></pre>
<p>✅ Right (Checking for content):</p>
<pre><code class="language-liquid">{% if my_string != blank %} This only shows if there is actual text. {% endif %}
</code></pre>
<p>✅ Right (Checking arrays/strings by size):</p>
<pre><code class="language-liquid">{% if my_array.size &gt; 0 %} This ensures the list isn't empty. {% endif %}
</code></pre>
