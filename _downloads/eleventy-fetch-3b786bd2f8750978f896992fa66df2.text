            <h3 id="attr-concat" tabindex="-1"><code>attr_concat</code> <a href="#attr-concat" aria-hidden="true" data-is-anchor="true">#</a></h3>
<p>A filter that concatenates values to an attribute array, returning a new object with the combined array. Useful for adding items to arrays like tags, classes, or other list-based attributes.</p>
<p><strong>Why use this?</strong> When working with objects that have array attributes (like tags), you often need to add additional values without mutating the original object. The <code>attr_concat</code> filter provides a clean way to combine existing array values with new ones, automatically handling duplicates.</p>
<p><strong>Features:</strong></p>
<ul>
<li>Non-mutating: Creates a new object, leaving the original unchanged</li>
<li>Automatically removes duplicates using Set</li>
<li>Handles multiple input types: arrays, JSON string arrays (killer feature for <code>.liquid</code>), or single values</li>
<li>Creates the attribute as an empty array if it doesn't exist</li>
<li>Logs an error if the existing attribute is not an array</li>
<li><code>TBC:</code> Supports nested attributes (e.g., <code>data.tags</code>)</li>
</ul>
<h4 id="example-add-tags-to-a-post-object-in-njk" tabindex="-1">Example: Add tags to a post object in <code>.njk</code>: <a href="#example-add-tags-to-a-post-object-in-njk" aria-hidden="true" data-is-anchor="true">#</a></h4>
<pre><code class="language-jinja2">{% set enhancedPost = post | attr_concat('tags', ['featured', 'popular']) %}

 * Concatenate values to an attribute array
 *
 * This function takes an object, an attribute name, and values to append.
 * It returns a new object with the attribute as a combined array of unique items.
 *
 * @param {Object} obj - The object to modify
 * @param {string} attr - The attribute name
 * @param {Array|string|*} values - Values to concatenate (array, JSON string array, or single value)
 * @returns {Object} A new object with the combined unique array
 */
export function attrConcat(obj, attr, values) {
  // Get the existing attribute value, default to empty array if not present
  const existingArray = obj?.[attr] || [];
<p>// Check if existing value is an array, convert if not
if (!Array.isArray(existingArray)) {
console.error(<code>attrConcat: Expected ${attr} to be an array, got ${typeof existingArray}</code>);
}</p>
<p>// Process the values argument
let valuesToAdd = [];
if (Array.isArray(values)) {
valuesToAdd = values;
} else if (typeof values === "string" && values.length >= 2 && <a href="http://values.at"><i><img src="https://www.google.com/s2/favicons?domain=values.at&sz=64"></i> values.at</a>(0) == "[" && <a href="http://values.at"><i><img src="https://www.google.com/s2/favicons?domain=values.at&sz=64"></i> values.at</a>(-1) == "]") {
// Try to parse as JSON array
try {
const parsed = JSON.parse(values);
if (Array.isArray(parsed)) {
valuesToAdd = parsed;
} else {
valuesToAdd = [values];
}
} catch {
// Not valid JSON, treat as single value
valuesToAdd = [values];
}
} else {
// If it's a single value, wrap it in an array
valuesToAdd = [values];
}</p>
<p>// Combine arrays and remove duplicates using Set
const combinedArray = [...new Set([...existingArray, ...valuesToAdd])];</p>
<p>// Return a new object with the combined array
return {
...obj,
[attr]: combinedArray,
};
}</p>
<p>/**</p>
<ul>
<li>attr_concat filter - Concatenate values to an attribute array</li>
<li></li>
<li>This filter takes an object, an attribute name, and values to append.</li>
<li>It returns a new object with the attribute as a combined array of unique items.</li>
<li></li>
<li>@param {Object} eleventyConfig - The Eleventy configuration object

            <h3 id="attr-includes" tabindex="-1"><code>attr_includes</code> <a href="#attr-includes" aria-hidden="true" data-is-anchor="true">#</a></h3>
<p>A filter that filters a list of items by checking if an attribute array includes a target value. Supports nested attribute names using dot notation.</p>
<p><strong>Why use this?</strong> When working with Eleventy collections, you often need to filter items based on tags or other array attributes in front matter. The <code>attr_includes</code> filter provides a flexible way to filter by any array attribute, with support for nested properties using dot notation.</p>
<h4 id="example-get-all-posts-that-include-javascript-tag" tabindex="-1">Example: Get all posts that include <code>#javascript</code> tag <a href="#example-get-all-posts-that-include-javascript-tag" aria-hidden="true" data-is-anchor="true">#</a></h4>
<pre><code data-caption="in .njk:" class="language-jinja2">{% set js_posts = collections.all | attr_includes('data.tags', '#javascript') %}

{% for post in js_posts %}
  &lt;h2&gt;{{ post.data.title }}&lt;/h2&gt;
{% endfor %}
</code></pre>

            <h3 id="fetch" tabindex="-1"><code>fetch</code> <a href="#fetch" aria-hidden="true" data-is-anchor="true">#</a></h3>
<p>A filter that fetches content from remote URLs or local files. For remote URLs, it uses <code>@11ty/eleventy-fetch</code> to download and cache files. For local paths, it reads files relative to the input directory.</p>
<p><strong>Why use this?</strong> When building static sites, you often need to include content from external sources or reuse content from local files. The <code>fetch</code> filter provides a unified way to retrieve content from both remote URLs and local files, with automatic caching for remote resources to improve build performance.</p>
<p><strong>Requirements:</strong> This filter requires the <code>@11ty/eleventy-fetch</code> package to be installed:</p>
<pre><code class="language-bash">npm install @11ty/eleventy-fetch
</code></pre>
<blockquote>
<p><code>NOTE:</code> If <code>@11ty/eleventy-fetch</code> is not installed, this filter will not be available. The plugin automatically detects whether the package is installed and only enables the filter if it's present.</p>
</blockquote>
<p><strong>Features:</strong></p>
<ul>
<li>Supports a URL (starting with <code>http://</code> or <code>https://</code>) or a local file path (relative to the input directory):
<ul>
<li><strong>Remote URLs</strong>: Downloads and caches content using <code>@11ty/eleventy-fetch</code>
<ul>
<li>Caches files for 1 day by default</li>
<li>Stores cached files in <code>[input-dir]/_downloads/</code> directory</li>
<li>Automatically revalidates after cache expires</li>
</ul>
</li>
<li><strong>Local files</strong>: Reads files relative to the Eleventy input directory
<ul>
<li>No caching needed for local files</li>
<li>Supports any file type that can be read as text</li>
</ul>
</li>
</ul>
</li>
<li><strong>Error handling</strong>: Throws descriptive errors if fetching fails</li>
<li><strong>Conditional loading</strong>: Only available when <code>@11ty/eleventy-fetch</code> is installed</li>
</ul>
<p><strong>Use Cases:</strong></p>
<ul>
<li>Fetch content from external APIs during build time</li>
<li>Include README files from GitHub repositories</li>
<li>Reuse content from local files across multiple pages</li>
<li>Download and inline external CSS or JavaScript</li>
<li>Fetch data from headless CMS or external data sources</li>
<li>Include shared content snippets without using Eleventy's include syntax</li>
</ul>
<blockquote>
<p><code>NOTE:</code> The filter returns raw text content. Use Eleventy's built-in filters like <code>| safe</code>, <code>| markdown</code>, or <code>| fromJson</code> to process the content as needed.</p>
</blockquote>
<p><strong>Examples:</strong></p>
<pre><code class="language-jinja2">{# Fetch and display remote content #}
{% set readme = &quot;https://raw.githubusercontent.com/user/repo/main/README.md&quot; | fetch %}
&lt;div class=&quot;readme&quot;&gt;
  {{ readme | markdown | safe }}
&lt;/div&gt;

{# Fetch JSON data from API #}
{% set data = &quot;https://api.example.com/data.json&quot; | fetch %}
{% set items = data | fromJson %}
{% for item in items %}
  &lt;p&gt;{{ item.title }}&lt;/p&gt;
{% endfor %}

{# Include local file content #}
{% set changelog = &quot;CHANGELOG.md&quot; | fetch %}
{{ changelog | markdown | safe }}

{# Fetch CSS from CDN and inline it #}
&lt;style&gt;
  {{ &quot;https://cdn.example.com/styles.css&quot; | fetch }}
&lt;/style&gt;

{# Reuse content across pages #}
{% set sharedContent = &quot;_includes/shared/footer.html&quot; | fetch %}
{{ sharedContent | safe }}
</code></pre>
<details class="breakout-item"><summary role="button" class="outline">How it works</summary>
```js
import EleventyFetch from "@11ty/eleventy-fetch";
import path from "path";
import fs from "fs/promises";
<p>/**</p>
<ul>
<li>
<p>fetch filter - Fetch a URL or local file and return its raw content</p>
</li>
<li></li>
<li>
<p>This filter takes a URL or local file path. For URLs, it downloads them</p>
</li>
<li>
<p>using eleventy-fetch to the input directory's _downloads folder.</p>
</li>
<li>
<p>For local paths, it reads them relative to the input directory.</p>
</li>
<li></li>
<li>
<p>@param {Object} eleventyConfig - The Eleventy configuration object
*/
export function fetchFilter(eleventyConfig) {
eleventyConfig.addFilter("fetch", async function (url) {
if (!url) {
throw new Error("fetch filter requires a URL or path");
}</p>
<p>// Get the input directory from Eleventy config
const inputDir = this.eleventy.directories.input;</p>
<p>// Check if it's a URL or local path
const isUrl = url.startsWith("http://") || url.startsWith("https://");</p>
<p>try {
if (isUrl) {
// Handle remote URLs with eleventy-fetch
const cacheDirectory = path.join(inputDir, "_downloads");
const content = await EleventyFetch(url, {
duration: "1d", // Cache for 1 day by default
type: "text", // Return as text
directory: cacheDirectory,
});
return content.toString(); // toString() handles bytes cache (inside eleventyComputed)
} else {
// Handle local file paths relative to input directory
const filePath = path.join(inputDir, url);
const content = await fs.readFile(filePath, "utf-8");
return content;
}
} catch (error) {
throw new Error(<code>Failed to fetch ${url}: ${error.message}</code>);
}
});
}</p>
</li>
</ul>

            <h3 id="if" tabindex="-1"><code>if</code> <a href="#if" aria-hidden="true" data-is-anchor="true">#</a></h3>
<p>An inline conditional/ternary operator filter that returns one value if a condition is truthy, and another if it's falsy. Similar to Nunjucks' inline if syntax, it is especially useful in <code>.liquid</code> templates.</p>
<p><strong>Features:</strong></p>
<ul>
<li>Returns <code>trueValue</code> if condition is truthy, otherwise returns <code>falseValue</code></li>
<li>Treats empty objects <code>{}</code> as falsy</li>
<li>Default <code>falseValue</code> is an empty string if not provided</li>
<li>Works with any data type for values</li>
</ul>
<p><strong>Examples:</strong> <!-- @TODO: better examples --></p>
<pre><code class="language-jinja2">{# Basic usage (defaults to empty string) #}
&lt;div class=&quot;{{ 'active' | if: isActive | default: 'inactive' }}&quot;&gt;Status&lt;/div&gt;

{# Toggle CSS classes #}
&lt;button class=&quot;{{ 'btn-primary' | if: isPrimary | default: 'btn-secondary' }}&quot;&gt;
  Click me
&lt;/button&gt;

{# Display different text #}
&lt;p&gt;{{ 'Online' | if: user.isOnline, 'Offline' }}&lt;/p&gt;

{# Use with boolean values #}
{% set isEnabled = true %}
&lt;div&gt;{{ 'Enabled' | if: isEnabled, 'Disabled' }}&lt;/div&gt;

{# Conditional attribute values #}
&lt;input type=&quot;checkbox&quot; {{ 'checked' | if: isChecked }}&gt;

{# With numeric values #}
&lt;span class=&quot;{{ 'has-items' | if: items.length }}&quot;&gt;
  {{ items.length }} items
&lt;/span&gt;

{# Chain with other filters #}
{% set cssClass = 'featured' | if: post.featured | upper %}
</code></pre>
<details class="breakout-item"><summary role="button" class="outline">How it works</summary>
```js
/**
 * if utility function - Ternary/conditional helper
 *
 * Returns trueValue if condition is truthy, otherwise returns falseValue.
 * Similar to Nunjucks' inline if: `value if condition else other_value`
 *
 * @param {*} trueValue - The value to return if condition is truthy
 * @param {*} condition - The condition to evaluate
 * @param {*} falseValue - The value to return if condition is falsy (default: empty string)
 * @returns {*} Either trueValue or falseValue based on condition
 */
export function iff(trueValue, condition, falseValue = "") {
  // Treat empty objects {} as falsy
  if (condition && typeof condition === "object" && !Array.isArray(condition) && Object.keys(condition).length === 0) {
    return falseValue;
  }
  return !!condition ? trueValue : falseValue;
}
<p>/**</p>
<ul>
<li>if filter - Inline conditional/ternary operator for templates</li>
<li></li>
<li>This filter provides a simple inline if/else similar to Nunjucks.</li>
<li></li>
<li>Usage in Liquid templates:</li>
<li>{{ "Active" | if: isActive, "Inactive" }}</li>
<li>{{ "Yes" | if: condition }}</li>
<li>{{ someValue | if: test, otherValue }}</li>
<li></li>
<li>@param {Object} eleventyConfig - The Eleventy configuration object
*/
export function ifFilter(eleventyConfig) {
eleventyConfig.addFilter("if", iff);
}</li>

            <h3 id="merge" tabindex="-1"><code>merge</code> <a href="#merge" aria-hidden="true" data-is-anchor="true">#</a></h3>
<p>A filter that merges arrays or objects together, similar to Twig's merge filter. For arrays, it concatenates them. For objects, it performs a shallow merge where later values override earlier ones.</p>
<p><strong>Why use this?</strong> When working with data in templates, you often need to combine multiple arrays or objects. The <code>merge</code> filter provides a clean way to merge data structures without writing custom JavaScript, making it easy to combine collections, merge configuration objects, or aggregate data from multiple sources.</p>
<p><strong>Examples:</strong> <!-- @TODO: better examples --></p>
<pre><code class="language-jinja2">{# Merge configuration objects #}
{% set defaultConfig = { theme: 'light', lang: 'en' } %}
{% set userConfig = { theme: 'dark' } %}
{% set finalConfig = defaultConfig | merge(userConfig) %}

{# Result: { theme: 'dark', lang: 'en' } #}
</code></pre>
<pre><code class="language-jinja2">{# Merge page metadata with defaults #}
{% set defaultMeta = {
  author: 'Site Admin',
  category: 'general',
  comments: false
} %}
{% set pageMeta = defaultMeta | merge(page.data) %}
</code></pre>
<details class="breakout-item"><summary role="button" class="outline">How it works</summary>
```js
/**
 * Merge objects together
 *
 * Shallow merges objects (later values override earlier ones)
 *
 * @param {Object} first - The first object
 * @param {...Object} rest - Additional objects to merge
 * @returns {Object} The merged result
 */
export function merge(first, ...rest) {
  // If first argument is null or undefined, treat as empty object
  if (first === null || first === undefined) {
    first = {};
  }
<p>// Only support objects
if (typeof first === "object" && !Array.isArray(first)) {
// Merge objects using spread operator (shallow merge)
return rest.reduce(
(acc, item) => {
if (item !== null && typeof item === "object" && !Array.isArray(item)) {
return { ...acc, ...item };
}
return acc;
},
{ ...first },
);
}</p>
<p>// If first is not an object, return empty object
return {};
}</p>

<li>This filter merges objects, similar to Twig's merge filter.</li>
<li></li>
<li>Usage in templates:</li>
<li>{{ obj1 | merge(obj2) }}</li>
<li>{{ obj1 | merge(obj2, obj3) }}</li>
<li></li>
<li>@param {Object} eleventyConfig - The Eleventy configuration object
*/
export function mergeFilter(eleventyConfig) {
eleventyConfig.addFilter("merge", merge);
}</li>

            <h3 id="remove-tag" tabindex="-1"><code>remove_tag</code> <a href="#remove-tag" aria-hidden="true" data-is-anchor="true">#</a></h3>
<p>A filter that removes a specified HTML element from provided HTML content. It removes the tag along with its content, including self-closing tags.</p>
<p><strong>Why use this?</strong> When working with content from external sources or user-generated content, you may need to strip certain HTML tags for security or presentation purposes. The <code>remove_tag</code> filter provides a simple way to remove unwanted tags like <code>&lt;script&gt;</code>, <code>&lt;style&gt;</code>, or any other HTML elements from your content.</p>
<p><strong>Features:</strong></p>
<ul>
<li>Removes both opening and closing tags along with their content</li>
<li>Handles self-closing tags (e.g., <code>&lt;br /&gt;</code>, <code>&lt;img /&gt;</code>)</li>
<li>Handles tags with attributes</li>
<li>Case-insensitive matching</li>
<li>Non-destructive: Returns new string, doesn't modify original</li>
</ul>
<p><strong>Security note:</strong> While this filter can help sanitize HTML content, it should not be relied upon as the sole security measure. For critical security requirements, use a dedicated HTML sanitization library on the server side before content reaches your templates.</p>
<h4 id="example-remove-all-script-tags-from-content" tabindex="-1">Example: Remove all script tags from content <!-- @TODO: better examples --> <a href="#example-remove-all-script-tags-from-content" aria-hidden="true" data-is-anchor="true">#</a></h4>

 *

export function removeTag(html, tagName) {
  if (!html || typeof html !== "string") {

<p>if (typeof tagName !== "string" || !tagName) {

}</p>

const regex = new RegExp(<code>&lt;${escapedTag}(?:\\s[^&gt;]*)?&gt;.*?&lt;\\/${escapedTag}&gt;</code>, "gis");
let result = html.replace(regex, "");</p>

const selfClosingRegex = new RegExp(<code>&lt;${escapedTag}(?:\\s[^&gt;]*)?\\s*\\/?&gt;</code>, "gi");

            <h3 id="section" tabindex="-1"><code>section</code> <a href="#section" aria-hidden="true" data-is-anchor="true">#</a></h3>
<p>A filter that extracts a named section from content marked with HTML comments. This is useful for splitting a single content file (like a Markdown post) into multiple parts that can be displayed and styled independently in your templates.</p>
<p><strong>Usage:</strong></p>
<ol>
<li>Mark sections in your content file (e.g., <code>post.md</code>):</li>
</ol>
<p>⚠️ <code>NOTE:</code> The <code>¡</code> symbol is used instead of <code>!</code> only to give examples below. Use <code>!</code> in your actual content files.</p>
<pre><code class="language-markdown"># My Post

&lt;!--section:intro--&gt;

This is the introduction that appears at the top of the page.

&lt;!--section:main--&gt;

This is the main body of the post with all the details.

&lt;!--section:summary,sidebar--&gt;

This content appears in both the summary and the sidebar!
</code></pre>
<ol start="2">
<li>Use the filter in your templates: <!-- @TODO: better examples --></li>
</ol>
<pre><code class="language-jinja2">{# Get the intro section #}
&lt;div class=&quot;page-intro&quot;&gt;
  {{ content | section('intro') | safe }}
&lt;/div&gt;

{# Get the main section #}
&lt;article&gt;
  {{ content | section('main') | safe }}
&lt;/article&gt;

{# Get the sidebar section #}
&lt;aside&gt;
  {{ content | section('sidebar') | safe }}
&lt;/aside&gt;
</code></pre>
<p><strong>Features:</strong></p>
<ul>
<li><strong>Multiple names</strong>: A single section can have multiple names separated by commas: <code>&lt;!--section:name1,name2--&gt;</code></li>
<li><strong>Case-insensitive</strong>: Section names are matched without regard to case</li>
<li><strong>Multiple occurrences</strong>: If a section name appears multiple times, the filter concatenates all matching sections</li>
<li><strong>Non-destructive</strong>: Returns extracted content without modifying the original input</li>
<li><strong>EOF support</strong>: Sections continue until the next <code>&lt;!--section*--&gt;</code> marker or the end of the file</li>
</ul>
<p><strong>Syntax Rules:</strong></p>
<ul>
<li>Sections start with: <code>&lt;!--section:NAME--&gt;</code> or <code>&lt;!--section:NAME1,NAME2--&gt;</code></li>
<li>Sections end at the next <code>&lt;!--section*--&gt;</code> marker or end of file</li>
<li>Whitespace around names and inside comments is automatically trimmed</li>
</ul>
<details class="breakout-item"><summary role="button" class="outline">How it works</summary>
```js
/**
 * Extract a named section from content marked with HTML comments
 *
 * @param {string} content - The content to process
 * @param {string} sectionName - The section name(s) to extract
 * @returns {string} The extracted section content
 */
export function section(content, sectionName) {
  if (!content || typeof content !== "string") {
    return content;
  }
<p>if (typeof sectionName !== "string" || !sectionName) {
return "";
}</p>
<p>// Normalize section name for comparison (trim whitespace)
const targetName = sectionName.trim().toLowerCase();</p>
<p>// Regex to match section markers with content up to the next section or end of string
// Captures: (1) section names, (2) content until next section marker or end
const sectionRegex = /<[!]--section:([^>]+)-->([\s\S]*?)(?=<[!]--section|$)/g;</p>
<p>let results = [];
let match;</p>
<p>// Find all sections
while ((match = sectionRegex.exec(content)) !== null) {
const namesStr = match[1];
const sectionContent = match[2];
const names = namesStr.split(",").map((n) => n.trim().toLowerCase());</p>
<p>// Check if any of the names match the target
if (names.includes(targetName)) {
results.push(sectionContent);
}
}</p>
<p>// Join all matching sections
return results.join("");
}</p>
<p>/**</p>
<ul>
<li>section filter - Extract a named section from content</li>
<li></li>
<li>Usage in templates:</li>
<li>{{ content | section('intro') }}</li>
<li>{{ content | section('footer') }}</li>
<li></li>
<li>Content format:</li>
<li>
<!--section:intro-->
</li>
<li>This is the intro content</li>
<li>
<!--section:main-->
</li>
<li>This is the main content</li>
<li>
<!--section:footer,sidebar-->
</li>
<li>This appears in both footer and sidebar sections</li>
<li></li>
<li>@param {Object} eleventyConfig - The Eleventy configuration object
*/
export function sectionFilter(eleventyConfig) {
eleventyConfig.addFilter("section", section);
}</li>
</ul>

            <h3 id="strip-tag" tabindex="-1"><code>strip_tag</code> <a href="#strip-tag" aria-hidden="true" data-is-anchor="true">#</a></h3>
<p>A filter that strips a specified HTML element from content while keeping its inner content intact. Only the opening and closing tags are removed; everything inside the tag is preserved in place.</p>
<p><strong>Why use this?</strong> When rendering HTML from a CMS or external source you sometimes need to unwrap a specific element (e.g. remove a wrapping <code>&lt;div&gt;</code> or <code>&lt;section&gt;</code>) without losing the content it contains. Unlike <code>remove_tag</code>, which discards the entire element and its content, <code>strip_tag</code> surgically removes only the tags themselves.</p>
<p><strong>Features:</strong></p>
<ul>
<li>Removes only the opening and closing tags — inner content is preserved</li>
<li>Handles tags with any attributes</li>
<li>Strips all occurrences of the tag, including nested ones</li>
<li>Case-insensitive matching</li>
<li>Non-destructive: Returns a new string, leaves the original unchanged</li>
</ul>
<h4 id="example-unwrap-a-wrapping-div-from-content" tabindex="-1">Example: Unwrap a wrapping <code>&lt;div&gt;</code> from content <a href="#example-unwrap-a-wrapping-div-from-content" aria-hidden="true" data-is-anchor="true">#</a></h4>
<pre><code class="language-jinja2">{% set unwrapped = htmlContent | strip_tag('div') %}

{{ unwrapped | safe }}
</code></pre>
<p>Input:</p>
<pre><code class="language-html">&lt;div class=&quot;wrapper&quot;&gt;
  &lt;p&gt;Hello&lt;/p&gt;
  &lt;p&gt;World&lt;/p&gt;
&lt;/div&gt;
</code></pre>
<p>Output:</p>
<pre><code class="language-html">&lt;p&gt;Hello&lt;/p&gt;
&lt;p&gt;World&lt;/p&gt;
</code></pre>
<details class="breakout-item"><summary role="button" class="outline">How it works</summary>
```js
/**
 * Strip a specified HTML element from provided HTML, keeping its inner content
 *
 * @param {string} html - The HTML content to process
 * @param {string} tagName - The tag name to strip (opening/closing tags removed, inner content kept)
 * @returns {string} The HTML with the specified tag stripped but its inner content preserved

  if (!html || typeof html !== "string") {
    return html;
  }
<p>if (typeof tagName !== "string" || !tagName) {

}</p>
<p>// Escape special regex characters in tag name

            <h3 id="unindent" tabindex="-1"><code>unindent</code> <a href="#unindent" aria-hidden="true" data-is-anchor="true">#</a></h3>
<details class="breakout-item"><summary role="button" class="outline">How it works</summary>
```js
/**
 * Remove the minimal common indentation from a multi-line string
 *
 * Finds the smallest leading-whitespace count across all non-empty lines
 * and strips that many characters from the beginning of every line.
 *
 * @param {string} content - The input string
 * @returns {string} The unindented string
 */
export function unindent(content) {
  const lines = String(content ?? "").split("\n");
  const minIndent = Math.min(...lines.filter((l) => l.trim()).map((l) => l.match(/^(\s*)/)[1].length));
  return lines.map((l) => l.slice(minIndent)).join("\n");
}

<li></li>
<li>Strips the smallest leading-whitespace indent shared by all non-empty</li>
<li>lines, useful for dedenting captured or indented template blocks.</li>
<li></li>
<li>Usage in templates:</li>
<li>{{ content | unindent }}</li>
<li></li>
<li>@param {Object} eleventyConfig - The Eleventy configuration object
*/
export function unindentFilter(eleventyConfig) {
eleventyConfig.addFilter("unindent", unindent);
